# Case Study: CRM & Billing System Data Integration

## 📌 Background
During my tenure as a Data Analyst in the Telecommunications sector, I identified a significant inefficiency in how subscriber data was handled between the **Customer Relationship Management (CRM)** and the **Billing systems**.

## ⚠️ The Problem
Staff were manually entering subscriber data into two separate systems. This led to:
* **High Error Rates:** Data mismatch between billing and customer profiles.
* **Operational Delay:** A 40% increase in manual processing time.
* **Customer Friction:** Delays in service activation.

## 🛠️ The Solution
I designed and implemented a **Mediation Data Layer** using SQL and Web Services to automate the synchronization.
* **Data Mapping:** Identified key fields (MSISDN, Subscriber ID, Plan Code) required for both systems.
* **SQL Automation:** Created stored procedures in Oracle SQL to extract, transform, and load (ETL) data in real-time.
* **Cross-Functional Collaboration:** Partnered with the Marketing and IT departments to ensure billing cycles aligned with new promotional campaigns.

## 📈 Results & Impact
* **Efficiency:** Reduced manual data entry time by **40%**.
* **Accuracy:** Improved data consistency between systems by **25%**.
* **Satisfaction:** Contributed to a **30% increase** in customer satisfaction scores due to faster service activation.

* **Database:** Oracle SQL, PL/SQL, Data Manipulation.
* **Business Analysis:** Requirement gathering, BPMN, and System Design.
